# SP2 — Two-Hand Musicality & Guidance: Design Spec

**Date:** 2026-06-28
**Status:** Draft
**Sub-project:** SP2 of 6 (see decomposition doc)
**Tracks:** A (Sound) + B (Surface)
**Depends on:** SP1 complete + two-hand dial foundation (track-a/track-b branches)
**Unlocks:** SP3 (Performance Shell)

---

## Goal

Make Froola sound richer, feel more musical, and guide new users through two-hand play.
SP1 proved the pipeline. SP2 makes it worth playing.

Three additions:
1. **Vibes** — 4 timbre presets selectable from a dropdown at the bottom of the canvas
2. **Warm-zone visual** — canvas background warmth reacts to chord quality (tension)
3. **Guided overlay** — first-session two-hand orientation, non-intrusive, dismissable

The two-hand dial paradigm (left hand = root note, right hand = chord quality) is already
built. SP2 wires it musically and adds expressiveness.

---

## Decisions locked

- **Vibes = timbre only.** No scale constraints — chromatic note selection stays fully open.
  Vibes change oscillator character + effects chain, not which notes are available.
- **Vibe picker = bottom-of-canvas dropdown**, always accessible during play. No gateway
  screen. No modal. The canvas is never hidden.
- **Warm zones driven by tension**, not by hand position. The background reacts to the chord
  quality the right hand has selected.

## L2 — Musical Mapper update (`src/engine/music/`)

**Owner: Track A**

### New signature

```typescript
// createMapper now accepts two optional signals instead of one
export function createMapper(
  vibe: Vibe
): (signals: GestureSignal[]) => MusicalCommand | null
```

The mapper reads `left` and `right` from the signals array:

```typescript
const left  = signals.find(s => s.handId === 'left');
const right = signals.find(s => s.handId === 'right');
```

### Two-hand mapping logic

**Left hand (root note):**
- `left.x` → index into `NOTES = ['A','B','C','D','E','F','G']`
- `left.y` → register offset (same ±12 semitone logic as SP1)
- If left is absent: hold last root note (do not emit null; keep previous chord sounding)

**Right hand (chord quality):**
- `right.x` → index into `QUALITIES = ['major','minor','maj7','min7','dom7','aug','dim']`
- `right.y` → texture (0 = sparse, 1 = dense — controls voice count or velocity in future SPs)
- If right is absent: hold last chord quality

**Combined → MusicalCommand:**
- `chord` — e.g. `"Cmaj7"` from root + quality
- `voicing` — root MIDI + quality intervals (see intervals table below)
- `register` — from `left.y` (0–1, maps to ±12 semitones from base)
- `texture` — from `right.y` (fixed 0.5 if right absent)
- `tension` — from quality's tension value (see table below)

### Quality intervals + tension table

| Quality | Intervals (semitones) | Tension |
|---|---|---|
| major  | [0, 4, 7]      | 0.0 |
| minor  | [0, 3, 7]      | 0.2 |
| maj7   | [0, 4, 7, 11]  | 0.1 |
| min7   | [0, 3, 7, 10]  | 0.3 |
| dom7   | [0, 4, 7, 10]  | 0.5 |
| aug    | [0, 4, 8]      | 0.7 |
| dim    | [0, 3, 6]      | 0.9 |

### Threshold (emit null when)

Both left and right are in the same state as last frame (same note index, same quality index,
and `y` shifts under threshold). Threshold values unchanged from SP1.

### Mouse fallback

Single `left`-hand signal only → right defaults to `major`. Chord = `[root]maj`. Tension = 0.0.

## L3 — Audio Engine update (`src/engine/audio/`)

**Owner: Track A**

### Vibe definitions

Four timbre presets. Each defines:
- Oscillator `type` (Web Audio OscillatorType)
- Reverb: wet/dry mix (0–1), decay (seconds)
- Delay: wet/dry, time (seconds), feedback (0–1)

| Vibe | Oscillator | Reverb wet | Reverb decay | Delay wet | Delay time | Delay fb |
|---|---|---|---|---|---|---|
| `warm`     | `sine`     | 0.3 | 2.5 | 0.0 | —    | —   |
| `bright`   | `triangle` | 0.1 | 1.0 | 0.15| 0.25 | 0.3 |
| `dark`     | `sine`     | 0.5 | 4.0 | 0.0 | —    | —   |
| `electric` | `square`   | 0.1 | 0.8 | 0.3 | 0.33 | 0.4 |

`square` and `sawtooth` are harsh at high volumes — compressor threshold should tighten
to `-12 dB` for `electric` to compensate.

### New method

```typescript
setVibe(vibe: Vibe): void
// Switches oscillator type + effects chain in real-time. No restart.
// Safe to call mid-play; effects crossfade within ~50ms.
```

### Effects chain (SP2)

```
OscillatorNode[0..2] → VoiceGains → MasterGain → Compressor → Reverb → Delay → Analyser → destination
```

Tone.js handles the Reverb and Delay nodes. Install: `npm install tone`.

Reverb = `Tone.Reverb`, Delay = `Tone.FeedbackDelay`. Both bypass when wet = 0.

### Interface additions (full diff vs SP1)

```typescript
class AudioEngine {
  // SP1 (unchanged)
  constructor()
  play(cmd: MusicalCommand): void
  getAnalyser(): AnalyserNode
  resume(): void
  suspend(): void

  // SP2 additions
  setVibe(vibe: Vibe): void
}
```

## L4 — Renderer update (`src/engine/renderer/`)

**Owner: Track B**

### New signature

```typescript
export function useRenderer(
  canvasRef: RefObject<HTMLCanvasElement>,
  signalsRef: RefObject<GestureSignal[]>,
  analyserRef: RefObject<AnalyserNode | null>,
  commandRef: RefObject<MusicalCommand | null>   // ← new: drives warm-zone color
): void
```

### Warm-zone visual system

Replace the static amber gradient with a dynamic one driven by `commandRef.current.tension`.

```typescript
// tension 0.0 = tonal → amber/gold
// tension 0.5 = moderate → orange
// tension 0.9 = dissonant → cool blue/purple

function tensionColor(tension: number): { inner: string; mid: string } {
  // lerp between amber (0.0) and indigo (1.0)
  // inner: rgba(245, 158, 11, α)  →  rgba(99, 102, 241, α)
  // mid:   rgba(217, 119,  6, α)  →  rgba(67,  56, 202, α)
}
```

Gradient radius: `Math.min(w, h) * (0.35 + tension * 0.15)` — expands slightly as tension rises.

If `commandRef.current` is null, fall back to static amber (same as SP1).

### Dial glow on audio amplitude

The selected item on each dial pulses opacity with amplitude. When amplitude > 0.2, scale the
selected label's alpha from 1.0 up to 1.5× (capped), giving a subtle brightness pulse.

## The Coordinator update (`src/coordinator.ts`)

**Co-owned.**

New responsibilities:
1. Hold `vibeRef` (current vibe, set from React UI state)
2. Update `commandRef` every time `play()` fires
3. Pass `commandRef` to `useRenderer`
4. Call `engine.setVibe()` when vibe changes

```typescript
export function useCoordinator(canvasRef: RefObject<HTMLCanvasElement | null>) {
  const [vibe, setVibe] = useState<Vibe>('warm');     // React state: discrete UI event
  const commandRef = useRef<MusicalCommand | null>(null);
  // ... (existing refs unchanged) ...

  // When vibe changes, update audio engine
  useEffect(() => {
    engineRef.current?.setVibe(vibe);
    mapperRef.current = createMapper(vibe);           // recreate mapper on vibe switch
  }, [vibe]);

  // In tick(): update commandRef after play()
  function tick() {
    const cmd = mapperRef.current(signalsRef.current);
    if (cmd) {
      commandRef.current = cmd;
      engineRef.current?.play(cmd);
    }
    rafId = requestAnimationFrame(tick);
  }

  useRenderer(canvasRef, signalsRef, analyserRef, commandRef);

  return { mode, requestCamera, useMouse, vibe, setVibe };  // expose setVibe to shell
}
```

## React shell additions (`src/components/`, `src/App.tsx`)

**Owner: Track B**

### Vibe dropdown

A single `<VibePicker>` component rendered over the canvas, anchored to the bottom center.

```
┌─────────────────────────────────────────┐
│                                         │
│            [canvas / dials]             │
│                                         │
│         ▼  warm          ← dropdown    │
└─────────────────────────────────────────┘
```

- Always visible during play (z-index above canvas)
- Collapsed by default: shows current vibe name + chevron
- Expanded: shows all 4 options as a list above the trigger
- Selecting a vibe collapses the list and calls `setVibe()`
- No page navigation, no modal overlay
- Styling: semi-transparent dark pill, monospace font, matches canvas aesthetic

```typescript
// src/components/VibePicker.tsx
type Props = { vibe: Vibe; onChange: (v: Vibe) => void };
```

### Guided first-play overlay

A `<GuidedOverlay>` rendered over the canvas on first session only.

**Trigger:** shown when `localStorage.getItem('froola-guided') !== 'done'`
**Dismiss conditions:** user clicks/taps anywhere, OR 15 seconds pass, OR both hands are detected
**On dismiss:** `localStorage.setItem('froola-guided', 'done')`

Content (two lines, centered, no blocking UI):

```
Left hand ←→  selects a note
Right hand ←→  selects a chord
```

Styling: white text, 40% opacity, large monospace font. Fades out over 0.5s on dismiss.
Does NOT block camera feed or canvas interaction.

## Done criteria

1. Moving both hands in front of the camera plays chords: root from left, quality from right
2. Selecting a vibe from the dropdown changes the sound character within ~50ms, no restart
3. All 4 vibes sound audibly distinct (sine/triangle/square wave + reverb differences)
4. Canvas background warmth changes with chord quality: major = amber, dim = cool blue
5. Guided overlay appears on first load and dismisses on touch/click or after 15s
6. Vibe dropdown is reachable during play without hiding the canvas
7. Mouse fallback: single left signal works (right defaults to major, tension = 0.0)
8. `npm run build` — zero TypeScript errors
9. `npm test` — all tests pass

## Performance constraints

- `setVibe()` must not cause an audible glitch or gap — effects must crossfade
- Warm-zone color interpolation: compute once per frame, not per-pixel
- `GuidedOverlay` is DOM, not canvas — no impact on rAF loop performance
