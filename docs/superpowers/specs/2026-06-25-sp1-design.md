# SP1 — Playable Core: Design Spec

**Date:** 2026-06-25
**Status:** Approved
**Sub-project:** SP1 of 6 (see decomposition doc)
**Tracks:** A (Sound — you) + B (Surface — cofounder)
**Depends on:** nothing
**Unlocks:** SP2, SP3 (parallel after interfaces frozen)

---

## Goal

Prove the full gesture → sound → visual pipeline end-to-end with minimum code.
A moving hand or mouse produces clean, click-free chord changes on a dark canvas.
No polish, no onboarding, no vibe system. Done when the pipeline works.

---

## Shared types (`src/engine/types.ts`)

Commit this file first — Track B is blocked on it.

```typescript
// Track B produces → Track A (L2) and L4 consume
export type GestureSignal = {
  x: number;        // 0–1, horizontal (left = 0, right = 1)
  y: number;        // 0–1, vertical (top = 0, bottom = 1)
  present: boolean; // is a hand/cursor actively tracked?
  handId: 'primary' | 'secondary';
};

// Track A (L2) produces → L3 and L4 consume
export type MusicalCommand = {
  chord: string;      // e.g. "Cmaj"
  voicing: number[];  // MIDI note numbers [root, third, fifth]
  register: number;   // 0–1, high to low
  texture: number;    // 0–1, sparse to dense (fixed 0.5 in SP1)
  tension: number;    // 0–1, tonal to dissonant (used by warm-zone renderer)
};
```

These types are co-owned. Do not change them without agreement from both tracks.

---

## Architecture

Four isolated layers wired by a thin coordinator. Layers do not import each other —
all communication is through the types above and the coordinator.

```
L1 (Track B) ─── GestureSignal ──→ Coordinator ──→ createMapper() [L2, Track A]
                                         │                  │
                                         │            MusicalCommand
                                         │                  ↓
                                         │         AudioEngine.play() [L3, Track A]
                                         │
                                    gestureRef + AnalyserNode
                                         ↓
                                   useRenderer() [L4, Track B]
```

**Hard rule:** all gesture → audio parameter updates flow through mutable `useRef`
values. React state is only for discrete UI mode changes. The coordinator never
calls `setState` in the hot path.

## L2 — Musical Mapping (`src/engine/music/`)

**Owner: Track A**

### Interface

```typescript
// createMapper is stateful (remembers last zone/register for threshold logic)
export function createMapper(vibe: string): (signal: GestureSignal) => MusicalCommand | null
```

Returns `null` when the gesture hasn't crossed a threshold — no audio update needed.

### Default vibe: Warm (C Major)

One hardcoded vibe in SP1. Vibe system comes in SP2.

**X-axis — 4 discrete snap zones:**

| Zone | x range | Chord | Function | Tension |
|---|---|---|---|---|
| 0 | 0.00–0.25 | Cmaj | I — Tonic | 0.0 |
| 1 | 0.25–0.50 | Fmaj | IV — Subdominant | 0.3 |
| 2 | 0.50–0.75 | Gmaj | V — Dominant | 0.6 |
| 3 | 0.75–1.00 | Cmaj | I — Tonic | 0.0 |

Zone 3 wraps back to I so the right edge feels resolved.

**Y-axis — register shift:**

- y = 0.0 (top): +12 semitones (one octave up from base)
- y = 0.5 (middle): base octave, root at C4 (MIDI 60)
- y = 1.0 (bottom): −12 semitones (one octave down)
- Register shift is linear; base voicing is [root, root+4, root+7] (major triad)

**Threshold — when `null` is returned:**

- x stays in the same snap zone, **and**
- y hasn't shifted by more than half a semitone (±0.5/12 ≈ 0.004 in y-space)

### `scales.ts` data shape

```typescript
type ChordSlot = {
  name: string;
  rootMidi: number;
  intervals: [0, number, number]; // always [root, third, fifth]

const WARM_MAJOR: ChordSlot[] = [
  { name: 'Cmaj', rootMidi: 60, intervals: [0, 4, 7], tension: 0.0 },
  { name: 'Fmaj', rootMidi: 65, intervals: [0, 4, 7], tension: 0.3 },
  { name: 'Gmaj', rootMidi: 67, intervals: [0, 4, 7], tension: 0.6 },

```

MIDI → Hz conversion: `440 * 2^((n - 69) / 12)`

## L3 — Audio Engine (`src/engine/audio/`)

**Owner: Track A**

### Interface

```typescript
export class AudioEngine {
  constructor()                    // creates AudioContext, wires effects chain, parks oscillators
  play(cmd: MusicalCommand): void  // ramps all 3 voices to new frequencies/gains
  getAnalyser(): AnalyserNode      // call once; store result in a ref
  resume(): void                   // call inside first user gesture handler
  suspend(): void                  // call on unmount
}
```

Instantiated once in the coordinator, stored in a `useRef`. Never recreated.

### Oscillator pool

Three persistent `OscillatorNode + GainNode` pairs, created at construction and
never stopped. Parked at 1 Hz (inaudible) until first `play()` call.

```
OscillatorNode[0] → GainNode[0] ─┐
OscillatorNode[1] → GainNode[1] ─┤→ MasterGainNode → CompressorNode → AnalyserNode → destination
OscillatorNode[2] → GainNode[2] ─┘
```

- Waveform: `sine` (default; other waveforms unlocked in SP5 P4)
- `MasterGainNode` value: `0.7`
- Per-voice gain: `0.7 / 3 ≈ 0.233`

### `play(cmd)` — per call

1. Convert `cmd.voicing` MIDI values to Hz
2. For each of the 3 voice nodes, schedule from `AudioContext.currentTime`:
   - `frequency.linearRampToValueAtTime(targetHz, now + 0.012)` — 12ms ramp
   - `gain.linearRampToValueAtTime(targetGain, now + 0.012)` — 12ms ramp
3. The 12ms ramp is the anti-click mechanism — imperceptible latency, no discontinuity

### Effects chain (SP1 — minimal)

- **DynamicsCompressorNode:** `threshold: -6, knee: 3, ratio: 4, attack: 0.003, release: 0.1`
  Acts as a safety limiter. Prevents clipping when all 3 voices peak.
- **AnalyserNode:** `fftSize: 256`. Exposed via `getAnalyser()` for Track B's renderer.
- No reverb/delay in SP1 — Tone.js effects chain added in SP2.

### AudioContext lifecycle

`AudioContext` is created in the constructor but browsers block audio until a user
gesture. `resume()` must be called from the first `pointerdown` or `keydown` on the
canvas. `play()` calls before `resume()` are silently ignored.

## The Coordinator (`src/coordinator.ts`)

**Co-owned. Track B writes it; Track A's exports must match the shapes below.**

```typescript
export function useCoordinator(canvasRef: RefObject<HTMLCanvasElement>) {
  const engineRef = useRef<AudioEngine | null>(null);
  const gestureRef = useRef<GestureSignal>({
    x: 0.5, y: 0.5, present: false, handId: 'primary'
  });

  useEffect(() => {
    engineRef.current = new AudioEngine();
    const resume = () => engineRef.current?.resume();
    window.addEventListener('pointerdown', resume, { once: true });
    return () => engineRef.current?.suspend();
  }, []);

  const signal = useGestureInput();           // L1 — Track B
  const mapperRef = useRef(createMapper('warm')); // L2 — Track A

  useEffect(() => {
    gestureRef.current = signal;
    if (!signal.present) return;
    const cmd = mapperRef.current(signal);
    if (cmd) engineRef.current?.play(cmd);    // L3 — Track A
  }, [signal]);

  useRenderer(                                // L4 — Track B
    canvasRef,
    gestureRef,
    engineRef.current?.getAnalyser() ?? null
  );
}
```

## Done criteria

SP1 is complete when all of the following are true:

1. Opening the app shows a dark canvas (no Vite boilerplate)
2. Camera permission prompt includes "Your camera never leaves your device."
3. Granting camera permission activates hand tracking within 3 seconds
4. Denying falls back to mouse mode, labeled clearly with a prompt to try camera
5. Moving hand/mouse left-to-right produces audibly different chords (I → IV → V → I)
6. Moving up/down shifts the pitch register
7. No audible clicks on chord transitions
8. No clipping when all 3 voices are active
9. Canvas renders a cursor orb that follows the hand/mouse
10. Orb glow scales with audio amplitude via AnalyserNode
11. `npm run build` passes with zero TypeScript errors

## Explicitly out of scope for SP1

Vibe picker, guided overlay, warm-zone gradient system, Tone.js effects, particle
system, landing page, onboarding, any premium feature.

## Performance constraints

- 30fps minimum with camera + audio + canvas simultaneously active
- MediaPipe inference must not block the audio thread — frame-skip under load
- `AudioContext` created/resumed only inside a user gesture handler
- Lighthouse Performance > 80 on desktop (measure before SP2, not after)

## Browser targets

- Chrome desktop — primary, must work
- Firefox desktop — secondary, verify before SP1 close
- Safari desktop — known AudioContext autoplay risk; document workarounds, do not
  block SP1 close on Safari
